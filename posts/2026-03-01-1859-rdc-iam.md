## 🆔 Destaques da Identidade (IAM)

### Entra ID Acelera Adoção de Passkeys
`ENTRA ID, PASSKEYS, FIDO2`

- **O Fato:** A partir de março de 2026, o Microsoft Entra ID ativará automaticamente perfis de passkey para inquilinos e priorizará o registro de passkeys sobre o Microsoft Authenticator.
- **O Ruído:** A migração de configurações padrão para inquilinos que não configurarem seus perfis.
- **O Sinal:** Esta é uma mudança estratégica da Microsoft para impulsionar a adoção de autenticação resistente a phishing em larga escala, solidificando as passkeys como o futuro da autenticação no ecossistema Entra.
- **🔑 Ação IAM:** Revise suas políticas de autenticação, prepare campanhas de conscientização e planeje a transição para passkeys como método primário.

### Microsoft Endurece Segurança do Entra Connect
`ENTRA ID, HYBRID IDENTITY, SECURITY`

- **O Fato:** Novas salvaguardas serão aplicadas no Microsoft Entra Connect a partir de março de 2026 para prevenir ataques de "SyncJacking" e tomada de contas de usuário.
- **O Ruído:** Detalhes técnicos sobre como a mitigação funciona internamente.
- **O Sinal:** Fortalecimento crítico da ponte híbrida entre on-prem e nuvem, mitigando um vetor de ataque que pode comprometer identidades sincronizadas. Essencial para a resiliência do seu ambiente híbrido.
- **🔑 Ação IAM:** Garanta que seu Microsoft Entra Connect esteja atualizado com a versão mais recente e configure as novas salvaguardas.

### Auth0 Aprimora Governança de Acesso a APIs
`AUTH0, API SECURITY, AUTHORIZATION`

- **O Fato:** As Políticas de Acesso à API para Aplicativos no Auth0 atingiram disponibilidade geral, permitindo controle granular sobre quais aplicativos podem solicitar tokens de acesso para APIs.
- **O Ruído:** A descrição da funcionalidade de controle granular.
- **O Sinal:** Essencial para implementar o princípio do menor privilégio em arquiteturas de API, garantindo que apenas aplicativos autorizados obtenham tokens com escopos específicos, fortalecendo a segurança de suas APIs.
- **🔑 Ação IAM:** Revise e implemente políticas de acesso à API para todos os seus aplicativos e APIs gerenciados pelo Auth0.

### Novo Phishing OAuth 2.0 Contorna MFA
`OAUTH, PHISHING, MFA BYPASS`

- **O Fato:** Uma campanha de phishing sofisticada abusa do fluxo OAuth 2.0 Device Authorization Grant para contornar a autenticação multifator (MFA) em contas Microsoft 365, roubando tokens de acesso e refresh.
- **O Ruído:** A percepção comum de que o maior risco SaaS está nas aplicações gerenciadas diretamente.
- **O Sinal:** Este ataque ressalta que a MFA não é uma bala de prata. Os atacantes estão mudando o foco para a camada de autorização (OAuth), explorando a confiança implícita em fluxos legítimos para obter acesso persistente e bypassar o MFA.
- **🔑 Ação IAM:** Audite permissões de aplicativos OAuth concedidas, implemente políticas de Acesso Condicional para fluxos de autorização e eduque usuários sobre consent phishing.

### Adoção de Passkeys Atinge Massa Crítica
`PASSKEYS, FIDO2, ADOÇÃO`

- **O Fato:** Em 1º de março de 2026, 96% dos dispositivos estão prontos para passkey, e 340 milhões de clientes bancários globais autenticam com passkeys ou FIDO2, um aumento de 180% ano a ano.
- **O Ruído:** O movimento para um sistema bancário sem senha e a taxa de adoção do JPMorgan Chase.
- **O Sinal:** A adoção em massa de passkeys valida a estratégia de migração para autenticação sem senha e resistente a phishing. O mercado está pronto e a experiência do usuário está melhorando, tornando a implementação uma prioridade estratégica.
- **🔑 Ação IAM:** Acelere seu roadmap de implementação de passkeys, priorizando casos de uso de alto impacto para usuários e clientes.

### Entra ID Reforça Segurança de Credenciais Móveis
`ENTRA ID, MOBILE SECURITY`

- **O Fato:** A Microsoft introduzirá a detecção de Jailbreak/Root para credenciais do Microsoft Entra no aplicativo Authenticator em fevereiro de 2026, impedindo o funcionamento em dispositivos comprometidos.
- **O Ruído:** A remoção automática de credenciais existentes nesses dispositivos.
- **O Sinal:** Aumenta a postura de segurança para autenticação móvel, mitigando o risco de credenciais serem exploradas em dispositivos comprometidos. Essencial para ambientes BYOD e para proteger acesso privilegiado via MFA.
- **🔑 Ação IAM:** Monitore ativamente a conformidade de dispositivos móveis e eduque usuários sobre os riscos de dispositivos com jailbreak/root.

### SailPoint Integra Governança ao AWS Security Hub
`SAILPOINT, IGA, AWS`

- **O Fato:** A SailPoint integrou-se ao "New Extended Plan for AWS Security Hub" em 26 de fevereiro de 2026.
- **O Ruído:** Detalhes específicos do "New Extended Plan".
- **O Sinal:** Consolida a visibilidade da postura de segurança de identidade e acesso na nuvem, permitindo que equipes de IGA e Cloud Security colaborem de forma mais eficaz e respondam a riscos de acesso em ambientes AWS.
- **🔑 Ação IAM:** Explore a integração para centralizar a governança de acesso e os insights de segurança em seus ambientes AWS.
